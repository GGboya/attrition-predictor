# 局限与未来工作

## 11.1 数据局限

1. **样本规模中等**：N = 5 469，正类仅 794。这一规模足以在内部做 CV + 测试集评估，但不足以支持深度架构（如 Transformer-Tabular）或极小子群（如"民办高校 × 女生"，N < 50）的可靠推断。
2. **单数据集、单时段**：数据来自一次横截面调研，缺乏时间序列；无法评估模型在"疫情后"、"经济下行期"等宏观冲击下的稳定性。
3. **自报偏差**：离职行为标签来自员工自报与后续跟踪访谈，可能存在"已离职员工不愿回应"带来的 missing-not-at-random 偏差。
4. **地域与行业代表性**：样本主要分布在长三角、珠三角与北京，制造业与 IT 业占比较高；对其他行业（如教育、医疗）与西部地区的泛化性待验证。

## 11.2 模型局限

1. **子群不均衡 AUC**：民办/独立高校毕业生（AUC = 0.659）与低收入组（AUC = 0.680）两个子群低于 0.70 阈值。这两个恰好是"正类率最高"的子群。未来工作需针对这类人群引入分层样本权重或专项模型。
2. **堆叠不可微 ↔ HRCF 代理性**：HRCF 以 MT-MLP 作为可微代理。附录 A 的一致性验证显示：全测试集 r = 0.922，但高风险区 Pearson r = 0.611，低于 0.85 的目标。我们的解读是：r 下降部分来自"range restriction"，因为高风险区的概率范围只有 [0.30, ~0.60]；但这仍意味着 HRCF 估计的 Δp 在迁移到堆叠时会有偏差。
3. **未对比外部基准**：本文有意未与 BORF 等外部论文数据做对比（其数据集不同，直接比较无意义），而是在同一切分上重训 7 个单体基线做对齐。未来若能获得外部 benchmark 数据集，应进行跨数据集外部验证。

## 11.3 干预局限

1. **成本函数为经验估计**：HRCF 的特征成本权重（1 Likert 级 ≈ ¥1.5k–3k）来自 HR 管理文献经验值，未经过具体组织的实证校准。部署时应与用人单位的 HR 预算机制对齐。
2. **Top-5 菜单不等于最终决策**：HRCF 给出的是建议菜单，最终是否实施、实施哪一条需要 HR 结合员工个案（是否愿意接受调岗、组织是否能提供相应资源）判断。
3. **未考虑组间交互**：当前 HRCF 独立为每个员工生成反事实，未考虑"一次性给全部门涨薪"这类共享干预的组合优化。这需要把 HRCF 扩展为组合优化问题（MIP / 强化学习）。

## 11.4 未来工作

- **纵向数据**：收集多期问卷或与 HR 系统打通，得到时间序列，支持 survival / Cox 模型。
- **跨组织外部验证**：与更多企业合作做 N ≥ 2 个机构的外部验证，评估迁移能力。
- **可微堆叠**：将 L2-LR + Isotonic 替换为单个 MLP 元学习器 + 温度缩放，让整个堆叠端到端可微，从而消除 HRCF 的代理性问题。
- **组合优化 HRCF**：将单员工反事实扩展为部门 / 团队层面的预算约束下的整体最优干预分配问题。
- **因果反事实**：当前 HRCF 是"预测翻转"意义下的反事实，未对应因果意义下的 intervention。引入结构因果模型（SCM）做因果识别是更严格的建模。
