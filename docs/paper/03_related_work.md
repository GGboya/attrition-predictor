# 相关工作

## 2.1 员工离职预测

传统员工离职预测主要沿着两条线发展。一条是基于统计方法的比例风险模型与 Logistic 回归，聚焦可解释系数，但在非线性与类别不平衡问题上表现有限。另一条是机器学习路线：早期以单一决策树、SVM 为主，近年转向树集成与深度表格模型。Liu 等人 2024 年提出的 BORF（Boosted Random Forest）在应届毕业生离职场景的 17k 数据集上报告 Sensitivity = 0.681、Accuracy = 0.786；但该工作未公开代码，且报告的指标之间数学上存在不一致（整体 Accuracy 0.786 与 AUC 0.69 难以同时成立）。

少数工作引入了 Stacking 或 Blending 集成以组合多个异质基模型，但这些工作大多未校准概率输出、未检验类别不平衡下 Sensitivity 的 bootstrap CI、也未对标签噪声做专门处理。

## 2.2 标签噪声与概率校准

Cleanlab（Northcutt 等）以 Confident Learning 框架估计每个训练样本的标签不可信度，并可输出用于降权或丢弃的样本级别权重。在问卷数据与医疗分类场景已验证其对 AUC 与 F1 的提升。

概率校准方面，Platt scaling 与 Isotonic 回归是最经典的两种后处理方法。在类别不平衡场景下，Isotonic 因不假设 sigmoid 形态而通常表现更稳定；我们在实验中也观察到 Isotonic 校准后 ECE 显著小于未校准版本（0.03 vs 0.09）。

## 2.3 反事实解释与可操作性

Wachter 等人 2017 年提出的反事实解释（Counterfactual Explanation，CF）回答的是"如果我把输入改成什么，模型预测就会翻转"的问题。后续 DiCE、Alibi、CoGS 等工具将其扩展到多样性（diversity）、稀疏性（sparsity）与可行性（feasibility）三个维度。

然而上述工作大多以"软约束"（在损失中加入惩罚项）方式处理可行性，最终方案未必满足约束。HR 场景下的约束通常是**硬约束**：性别、高校类型这类人口学变量不可改变；收入只能升高；工作压力只能降低；Likert 问卷题目取值必须为整数或 0.25 步长。HRCF 针对这一场景设计"投影式梯度下降"，在每一步强制满足所有硬约束，从而保证所有返回的方案都是可执行的。

## 2.4 本文定位

本文与已有文献的区别在于：（a）将"噪声稳健的堆叠预测 + 硬约束反事实干预"作为一个闭环系统呈现；（b）在同一切分上用统一协议重训七种单体基线并做 DeLong 配对检验，而非引用外部论文的指标；（c）针对应届毕业生这一样本规模中等（5k）但应用价值高的群体进行完整评估。
