# 讨论

## 10.1 关键发现的管理学解读

**一、预测端给出的"离职驱动画像"**

SHAP 分析显示，Top-5 特征中三个属于**结构性变量**（工作单位性质、高校类型、收入水平），两个属于**感知变量**（工作满意度、工作匹配度）。这意味着：

- 模型学到的第一层判别是"这是什么样的人在什么样的单位做什么样的工资"——结构性变量。HR 经理对此类变量几乎无干预能力，但可以在招聘阶段将其纳入风险评估。
- 第二层判别才是员工对工作的主观评价。这两个变量**都是可干预的**，正是 HRCF 的主要作用对象。

尤其值得注意的是，本研究训练时**没有使用离职意向**（自报的 1–5 级 Likert）作为特征。也就是说，模型仅凭客观工作环境与工作评价就达到了 AUC = 0.78。实际部署时，如果组织可以收集员工的离职意向问卷，信号应会进一步加强；但即使不便收集，本方法依然有实用价值。

**二、干预端的经济合理性**

HRCF 给出的 Top-5 干预菜单平均成本 ¥13.9k、Δp = 0.31，即单位成本降低 1% 离职概率约 ¥418。粗略估算，如果一位员工实际离职会给企业造成约 ¥80k 的重置成本（招聘 + 培训 + 机会成本，Liu 等 2024 调研数据），那么对高风险员工实施 ¥14k 的干预期望收益为 $0.31 \times 80\text{k} - 14\text{k} \approx 11\text{k}$——这是正 ROI 的，说明系统具有经济合理性。

当然，这只是平均估算；HR 经理真正需要的是菜单式选择。HRCF 返回的 Top-5 方案（Section 8.3）允许 HR 根据预算、员工重要性与干预可行性挑选其中一条或组合实施。

## 10.2 为何堆叠优于单模型

单纯从 AUC 看，冠军仅比 RF 高 0.0035（p = 0.498，DeLong 不显著）。这似乎说明"堆叠没必要"。但仔细看：

- 冠军在 5/7 基线上显著更优；RF 是唯一例外。
- 冠军的 RF 基学习器本身在元系数占 0.376，已经捕获 RF 能提供的主要信号。
- 真正让冠军不同的是 MT-MLP（coef 0.316）与 ET（coef 0.284）的互补贡献，使得整体在 **PR-AUC、Bal-Acc、校准** 三方面都不弱于单 RF。
- 在**下游 HRCF 干预**中，冠军的 MT-MLP 部件提供了可微性，这是 RF 单独做不到的。

因此，堆叠的真正价值不在"单独 AUC 数字"，而在于**把一组互补的能力组合在同一个校准概率上**：排序能力 + 可微性 + 稳健性，这是任何一个单体基线都不能同时提供的。

## 10.3 对研究者的启示

**三个可以复用的工程选择**：

1. **Cleanlab v6 二值权重**：在 Likert / 问卷类标签噪声场景下，二值权重 (1.0/0.3) 比连续权重更稳定、比"丢弃"更安全。只要基学习器支持 `sample_weight` 参数即可无缝接入。
2. **Isotonic 在不平衡数据上优于 Platt**：当类别不平衡 + stack 元输出分布非 sigmoid 形态时，Isotonic 在 ECE / Brier 上明显占优。
3. **HRCF 投影级联的可迁移性**：HRCF 的设计模式（投影式梯度下降 + snap-to-grid + 多重启动 + 贪心多样性选择）对任何带有"混合离散 / 连续 / 不可变 / 单调 / 有界"约束的表格分类问题都适用，不只限于离职预测。
